## 질문 섹션

### Q1: 관리자 계정 관리 방식
관리자 계정은 어떻게 관리되나요?

A) 시드 데이터로 사전 생성 (변경 불가)
B) 시드 데이터로 사전 생성 (관리자가 비밀번호 변경 가능)
C) 관리자 등록 API 제공 (동적 생성)
D) 슈퍼 관리자가 관리자 계정 생성/삭제
E) Other (설명해주세요)

[Answer]:B

### Q2: 로그인 시도 제한 정책
로그인 시도 제한은 어떻게 적용되나요?

A) IP 기반 제한 (5회 실패 시 30분 차단)
B) 계정 기반 제한 (5회 실패 시 30분 차단)
C) IP + 계정 조합 제한 (5회 실패 시 30분 차단)
D) 제한 없음 (단순 로그 기록만)
E) Other (설명해주세요)

[Answer]:B

### Q3: 테이블 비밀번호 정책
테이블 태블릿의 비밀번호는 어떤 형식이어야 하나요?

A) 4자리 숫자 PIN (예: 1234)
B) 6자리 이상 영숫자 조합
C) 8자리 이상 영숫자 + 특수문자
D) 관리자가 자유롭게 설정 (최소 길이만 제한)
E) Other (설명해주세요)

[Answer]:A

### Q4: JWT 토큰 저장 위치
JWT 토큰은 클라이언트에서 어디에 저장되나요?

A) localStorage (XSS 취약, 간단)
B) sessionStorage (탭 닫으면 삭제)
C) HttpOnly Cookie (XSS 방어, CSRF 고려 필요)
D) 메모리만 (새로고침 시 재로그인)
E) Other (설명해주세요)

[Answer]:A

### Q5: 토큰 갱신(Refresh) 정책
16시간 토큰 만료 후 처리 방식은?

A) 만료 시 즉시 로그아웃 (재로그인 필요)
B) Refresh Token 제공 (자동 갱신)
C) 만료 30분 전 자동 갱신
D) 사용자 활동 감지 시 자동 연장
E) Other (설명해주세요)

[Answer]:D

### Q6: 다중 매장 인증 격리
다중 매장 환경에서 인증은 어떻게 격리되나요?

A) 매장ID를 JWT 페이로드에 포함 (모든 API 요청 시 검증)
B) 매장별 별도 데이터베이스 스키마
C) 매장ID를 URL 경로에 포함 (/api/stores/{storeId}/...)
D) 매장ID를 헤더에 포함 (X-Store-Id)
E) Other (설명해주세요)

[Answer]:A

### Q7: 테이블 세션 자동 시작 조건
테이블 세션은 언제 자동으로 시작되나요?

A) 테이블 태블릿 로그인 시 즉시 세션 생성
B) 첫 주문 생성 시 세션 자동 생성
C) 관리자가 "세션 시작" 버튼 클릭 시
D) 테이블 초기 설정 완료 시 세션 생성
E) Other (설명해주세요)

[Answer]:A

### Q8: 로그인 실패 시 에러 메시지
로그인 실패 시 어떤 에러 메시지를 표시하나요?

A) 구체적 메시지 ("비밀번호가 틀렸습니다", "존재하지 않는 계정입니다")
B) 일반적 메시지 ("로그인 정보가 올바르지 않습니다")
C) 보안 강화 메시지 ("인증 실패. 관리자에게 문의하세요")
D) 시도 횟수 포함 메시지 ("로그인 실패. 남은 시도: 3회")
E) Other (설명해주세요)

[Answer]:A

### Q9: 테이블 태블릿 자동 로그인 실패 처리
localStorage에 저장된 정보로 자동 로그인 실패 시 어떻게 처리하나요?

A) localStorage 정보 삭제 후 로그인 화면 표시
B) 에러 메시지 표시 후 재시도 버튼 제공
C) 관리자에게 알림 전송 (SSE)
D) 로그 기록만 하고 로그인 화면 표시
E) Other (설명해주세요)

[Answer]:B

### Q10: 비밀번호 변경 기능
관리자 또는 테이블 비밀번호 변경 기능이 필요한가요?

A) 관리자 비밀번호 변경 기능 제공 (로그인 후)
B) 테이블 비밀번호 변경 기능 제공 (관리자만)
C) 둘 다 제공
D) 둘 다 제공하지 않음 (시드 데이터 고정)
E) Other (설명해주세요)

[Answer]:A

### Q11: JWT 페이로드 구성
JWT 토큰에 어떤 정보를 포함해야 하나요?

A) 최소 정보 (userId, storeId, role, exp)
B) 확장 정보 (userId, storeId, role, tableId, sessionId, exp)
C) 사용자 정의 (필요한 정보 나열해주세요)
D) 표준 클레임만 (sub, iss, exp, iat)
E) Other (설명해주세요)

[Answer]:B

### Q12: 동시 로그인 제한
같은 계정으로 여러 곳에서 동시 로그인이 가능한가요?

A) 제한 없음 (여러 기기에서 동시 로그인 가능)
B) 단일 세션만 허용 (새 로그인 시 이전 세션 무효화)
C) 최대 N개 세션 허용 (예: 3개)
D) 관리자는 단일, 테이블은 제한 없음
E) Other (설명해주세요)

[Answer]:D
