# Unit 2: 인증 모듈 - Functional Design Clarification

## CQ1: 테이블 세션 시작 시점 충돌
Q7에서 "A) 태블릿 로그인 시 즉시 세션 생성"을 선택하셨습니다.
그런데 기존 요구사항(US-A07)에서는 "첫 주문 생성 시 세션 자동 생성"으로 정의되어 있습니다.

어떤 방식으로 진행할까요?

A) Q7 답변 유지 - 태블릿 로그인 시 즉시 세션 생성 (US-A07 스토리 수정 필요)
B) 기존 요구사항 유지 - 첫 주문 생성 시 세션 자동 생성 (Q7 답변 변경)
C) 하이브리드 - 로그인 시 "대기" 세션 생성, 첫 주문 시 "활성" 상태로 전환
E) Other (설명해주세요)

[Answer]:B

## CQ2: 구체적 에러 메시지의 보안 리스크
Q8에서 "A) 구체적 메시지"를 선택하셨습니다.
"존재하지 않는 계정입니다"와 같은 메시지는 공격자에게 유효한 계정 존재 여부를 노출할 수 있습니다.

이 보안 리스크를 인지하고 진행하시겠습니까?

A) 그대로 진행 - 사용성 우선 (구체적 메시지 유지)
B) 보안 우선으로 변경 - 일반적 메시지 ("로그인 정보가 올바르지 않습니다")
C) 관리자는 구체적 메시지, 테이블은 일반적 메시지
E) Other (설명해주세요)

[Answer]:B
