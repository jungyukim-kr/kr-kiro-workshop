# NFR Design Plan - Unit 4: 고객 프론트엔드

## Unit Context
- **Unit**: unit-4-customer-fe
- **기술 스택**: Vanilla JS + Bootstrap 5 CDN
- **배포**: Spring Boot static/
- **NFR 특성**: Frontend 정적 파일, 별도 인프라 컴포넌트 없음

## Plan Steps

- [x] 1. NFR Requirements 산출물 분석
- [x] 2. NFR Design Patterns 문서 생성 (Frontend 패턴)
- [x] 3. Logical Components 문서 생성 (모듈 구조)

## 질문

Frontend 정적 파일 프로젝트 특성상 추가 질문 없이 진행합니다.
- 인프라 컴포넌트 (큐, 캐시, DB) 해당 없음
- 확장성/복원력 패턴은 Backend(Unit 2, 3)에서 처리
- Frontend 패턴은 NFR Requirements 답변으로 충분히 결정됨
