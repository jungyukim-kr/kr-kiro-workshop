# Security Test Instructions

## 목적

인증 모듈의 보안 요구사항 (NFR-AUTH-02) 충족 여부를 검증합니다.

## 보안 테스트 체크리스트

### 1. JWT 토큰 보안

| # | 테스트 항목 | 검증 방법 | 기대 결과 |
|---|-----------|-----------|-----------|
| S-01 | 만료된 토큰 거부 | 만료된 토큰으로 API 호출 | 401 TOKEN_EXPIRED |
| S-02 | 변조된 토큰 거부 | payload 수정 후 API 호출 | 401 TOKEN_INVALID |
| S-03 | 다른 키로 서명된 토큰 거부 | 다른 secret으로 생성한 토큰 사용 | 401 TOKEN_INVALID |
| S-04 | 토큰 없이 보호 API 접근 불가 | Authorization 헤더 없이 요청 | 401 TOKEN_MISSING |

### 2. 비밀번호 보안

| # | 테스트 항목 | 검증 방법 | 기대 결과 |
|---|-----------|-----------|-----------|
| S-05 | 비밀번호 평문 저장 안 됨 | DB admin 테이블 password_hash 확인 | bcrypt 해시 형식 ($2a$...) |
| S-06 | 비밀번호 변경 시 현재 비밀번호 확인 | 잘못된 currentPassword로 변경 시도 | 401 AUTHENTICATION_FAILED |
| S-07 | 비밀번호 변경 후 기존 토큰 무효화 | 변경 후 이전 토큰으로 요청 | 401 TOKEN_INVALID |

### 3. 접근 제어

| # | 테스트 항목 | 검증 방법 | 기대 결과 |
|---|-----------|-----------|-----------|
| S-08 | TABLE 역할로 ADMIN API 접근 불가 | TABLE 토큰으로 /admin/** 접근 | 403 |
| S-09 | ADMIN 역할로 TABLE API 접근 불가 | ADMIN 토큰으로 /customer/** 접근 | 403 |
| S-10 | 다른 매장 리소스 접근 불가 | storeId=1 토큰으로 /stores/2/** 접근 | 403 STORE_ACCESS_DENIED |

### 4. 입력 검증

| # | 테스트 항목 | 검증 방법 | 기대 결과 |
|---|-----------|-----------|-----------|
| S-11 | SQL Injection 방어 | storeCode에 `' OR 1=1 --` 입력 | 401 (정상 검증 실패) |
| S-12 | XSS 방어 | username에 `<script>alert(1)</script>` 입력 | 401 (정상 검증 실패) |
| S-13 | 빈 필드 거부 | 필수 필드 누락 | 400 VALIDATION_FAILED |
| S-14 | PIN 형식 검증 | 4자리 숫자가 아닌 값 | 400 VALIDATION_FAILED |

### 5. 에러 메시지 보안

| # | 테스트 항목 | 검증 방법 | 기대 결과 |
|---|-----------|-----------|-----------|
| S-15 | 매장 미존재 시 일반 에러 | 없는 매장 코드로 로그인 | "로그인 정보가 올바르지 않습니다" (매장 미존재 노출 안 됨) |
| S-16 | 계정 미존재 시 일반 에러 | 없는 username으로 로그인 | "로그인 정보가 올바르지 않습니다" (계정 미존재 노출 안 됨) |
| S-17 | 비밀번호 불일치 시 일반 에러 | 잘못된 비밀번호로 로그인 | "로그인 정보가 올바르지 않습니다" (비밀번호 불일치 노출 안 됨) |

- S-11~S-14는 수동 테스트 또는 OWASP ZAP 등 도구 활용 권장
- S-15~S-17은 unit test (TC-AUTH-010~012)에서 이미 검증됨
