# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 신규 구축
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI + 관리자 UI + 백엔드 서버 + DB)
- **Complexity**: Complex (실시간 통신, 인증, 세션 관리, 다중 매장 지원)

---

| 항목 | 결정 |
|------|------|
| 백엔드 | Java (Spring Boot) |

## 기능 요구사항 (Functional Requirements)

### FR-1: 고객용 기능

#### FR-1.1: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호, 비밀번호)
- 로그인 정보 localStorage에 저장
- 저장된 정보로 자동 로그인

#### FR-1.2: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지, ==맵기 수준 (관리자가 설정한 경우 표시)==
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-1.3: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 임시 저장 (localStorage, 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-1.4: 주문 생성
- 주문 내역 최종 확인 후 확정
- ==메뉴별 요청사항 입력 기능 (자유 텍스트, 예: "청양고추 빼주세요")==
- ==메뉴별 맵기 옵션 선택 (관리자가 메뉴에 맵기 옵션을 설정한 경우, 고객이 주문 시 선택)==
- 성공 시: 주문 번호 표시, 장바구니 비우기, 5초 후 메뉴 화면 리다이렉트
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가, ==요청사항, 맵기 옵션==), 총 금액, 세션 ID

#### FR-1.5: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- ==주문 상태 실시간 업데이트 (선택사항)==
- 현재 테이블 세션 주문만 표시
- 매장 이용 완료 처리된 주문 제외
- ==페이지네이션 또는 무한 스크롤==

### FR-2: 관리자용 기능

#### FR-2.1: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한

#### FR-2.2: 실시간 주문 모니터링
- SSE(Server-Sent Events) 기반 실시간 주문 업데이트
- 테이블별 그리드/카드 레이아웃
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- ==주문별 상세: 테이블 번호, 주문 번호, 주문 시각, 주문 메뉴 및 수량 (축약), 총 금액==
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- ==신규 주문 시각적 강조 (색상 변경, 애니메이션)==
- 2초 이내 주문 표시
- ==테이블별 필터링 기능==

#### FR-2.3: 테이블 관리
- **테이블 태블릿 초기 설정**: 번호, 비밀번호 설정, 16시간 세션 생성, 자동 로그인 활성화
- **주문 삭제**: 확인 팝업 표시, 즉시 삭제, 총 주문액 재계산, ==성공/실패 피드백==
- **테이블 세션 관리**:
  - ==세션 시작: 해당 테이블의 첫 주문 시 자동으로 새 세션 시작==
  - 세션 종료 (이용 완료): ==확인 팝업 표시==, 주문 내역을 과거 이력(==OrderHistory==)으로 이동, 주문 목록/총액 리셋, ==성공/실패 피드백==
  - ==새 고객이 이전 주문 내역 없이 시작 가능==
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록(시간 역순), 각 주문에 주문 번호/시각/메뉴 목록/총 금액/==매장 이용 완료 시각== 표시, 날짜 필터링, =="닫기" 버튼으로 대시보드 복귀==
- **데이터 관리**: ==OrderHistory 테이블에 저장, 세션 ID로 주문 그룹화, 완료 시각 기록==

#### FR-2.4: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지 URL, ==맵기 수준, 맵기 옵션 목록==)
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증
- ==맵기 수준: 선택적 입력 (예: 안매움/약간매움/매움/아주매움)==
- ==맵기 옵션: 메뉴별로 고객이 선택 가능한 맵기 옵션 목록 설정 (선택적)==

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능
- SSE 기반 실시간 통신, 주문 표시 2초 이내
- 터치 반응 즉시 피드백

### NFR-2: 보안
- JWT 토큰 기반 인증
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

### NFR-3: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 레이아웃
- 명확한 시각적 계층 구조

### NFR-4: 데이터 무결성
- 주문 데이터 서버 측 검증
- 세션 기반 주문 그룹화
- 이력 데이터 보존

## 제외 범위 (Out of Scope)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 업로드/리사이징
- 알림 시스템 (푸시, SMS, 이메일, 소리)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석 대시보드, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)
- ==컨텐츠 관리 시스템, 광고 기능==
